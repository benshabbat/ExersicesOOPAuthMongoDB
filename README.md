## 🔑 עקרונות מרכזיים ב-OOP

### 1. Encapsulation (הסתרה)
הסתרת מידע פנימי וחשיפת רק מה שצריך:
```javascript
class BankAccount {
    #balance; // Private
    
    get balance() {
        return this.#balance;
    }
}
```

### 2. Inheritance (הורשה)
יצירת מחלקות חדשות ממחלקות קיימות:
```javascript
class Animal {
    eat() { }
}

class Dog extends Animal {
    bark() { }
}
```

### 3. Polymorphism (פולימורפיזם)
אובייקטים שונים מגיבים אחרת לאותה מתודה:
```javascript
class Shape {
    getArea() { return 0; }
}

class Circle extends Shape {
    getArea() { return Math.PI * this.radius ** 2; }
}

class Rectangle extends Shape {
    getArea() { return this.width * this.height; }
}
```

### 4. Abstraction (הפשטה)
הסתרת מורכבות וחשיפת ממשק פשוט:
```javascript
class Calculator {
    // מתודות פנימיות מורכבות
    #complexCalculation() { }
    
    // ממשק פשוט למשתמש
    calculate(a, b) {
        return this.#complexCalculation();
    }
}
```

## 📖 מילון מונחים

- **Class (מחלקה)**: תבנית ליצירת אובייקטים
- **Object (אובייקט)**: מופע של מחלקה
- **Constructor (קונסטרקטור)**: פונקציה שרצה בעת יצירת אובייקט
- **Method (מתודה)**: פונקציה בתוך מחלקה
- **Property (מאפיין)**: משתנה בתוך מחלקה
- **this**: הפניה לאובייקט הנוכחי
- **extends**: יורש ממחלקת אב
- **super**: קריאה למחלקת האב
- **static**: מתודה/מאפיין שייכים למחלקה ולא למופע
- **private (#)**: מאפיין פרטי, לא נגיש מחוץ למחלקה
- **getter/setter**: מתודות מיוחדות לקריאה/כתיבה של מאפיינים
